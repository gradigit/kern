1. merge the release-prep PR
2. tag `v0.1.3`
3. rebuild the final release artifacts from the tagged commit

5. run `./scripts/verify-github-release-asset.sh v0.1.3`

7. update this file with final publish time, final asset digests, and published

proves same-release asset integrity only; it does not authenticate publisher
identity the way Apple signing and notarization would.
